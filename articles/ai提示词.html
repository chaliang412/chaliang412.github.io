LaTeX

请用 LaTeX 生成实现以下功能的代码： 在编写代码时，必须严格遵守以下规范：

  1. 【宏包精简】仅加载实现功能所必需的宏包（如数学用 amsmath，绘图用 tikz），严禁无意义地堆砌不相关的宏包，严禁使用已被淘汰的旧宏包（如 a4wide、epsfig）。
  2. 【环境隔离】数学公式绝对禁止在正文中直接硬敲（如 $E=mc^2$），必须严格套用在数学环境中（行内用 $...$，行间用 \[...\] 或 equation 环境）。
  3. 【图文浮动体】所有图片（figure）和表格（table）严禁放在正文段落之间硬编码，必须使用浮动体环境包裹，并严格配对 \caption{} 和 \label{}（标签必须在标题下方）。
  4. 【交叉引用】严禁在正文中手动输入硬编码的编号（如“见图1”、“由公式(2)可知”），必须使用 \ref{} 或 \eqref{} 进行动态引用，防止增删内容后编号全盘错乱。
  5. 【三线表规范】所有数据表格严禁使用 \hline 画出全封闭的网格线（Excel风格），必须使用 booktabs 宏包提供的 \toprule、\midrule、\bottomrule 绘制标准学术三线表。
  6. 【中文排版规范】使用 ctexart 或 ctexbook 文档类时，严禁在导言区多余地加载 CJK 或 fontspec 宏包；中英文混排时，必须在中文与英文/数字之间用半角空格隔开（如 使用 Python 进行计算）。
  7. 【标点规范】双引号用``''，即Tab上边和右ENTER按键左边的符号，且中文句号全部改为英文的。

如无特殊说明，以上要求在本次对话中一直遵循。

Mathematica

请用 Mathematica (Wolfram Language) 编写代码实现如下效果： 在编写代码时，必须严格遵守以下“函数式编程”规范：

  1. 【禁止传统循环】严禁使用 For、While、Do 等传统命令式循环。必须改用 Map (/@)、Apply (@@)、Table、Array 或递归等函数式构造。
  2. 【善用简写符号】鼓励大量使用 @、//、~、@@ 等后缀和中缀简写形式来缩减代码体积（如用 f/@list 代替 Map[f, list]，用 list//f 代替 f[list]）。
  3. 【极简注释】Mathematica 本身具有高度自描述性，严禁在代码内部添加任何行内注释(* ... *)，保持代码像数学公式一样纯净。
  4. 【内置函数优先】优先使用内置的列表处理函数（如 Select、Cases、Pick、SortBy、GatherBy）代替手动遍历。
  5. 【匿名函数】鼓励使用纯函数（# 与 &）代替定义冗余的独立小函数（如用 #^2 & 代替 f[x_]:=x^2）。

如无特殊说明，以上要求在本次对话中一直遵循。

Python

请用 Python 编写代码（必要时注明依赖库）实现以下效果： 在编写代码时，必须严格遵守以下规则：

  1. 【类型标注】所有函数定义必须包含完整的参数类型标注和返回值类型标注（如 def add(a: int, b: int) -> int:），严禁编写无类型提示的“裸函数”。
  2. 【异常处理】所有可能失败的外部操作（文件 I/O、网络请求、数据库查询、JSON 解析）必须使用 try...except 包裹，并明确捕获具体异常类型（如 FileNotFoundError、json.JSONDecodeError），严禁使用裸 except: 吞掉所有错误。
  3. 【路径兼容】所有文件路径必须使用 pathlib.Path 构造，严禁使用字符串拼接（如 "folder/" + name）。路径拼接用 Path("folder") / name，使代码天然跨平台兼容 Windows 和 Linux。
  4. 【虚拟环境与依赖】如果代码需要第三方库，必须在代码首部以注释形式列出对应的 pip install 命令（如 # pip install requests pandas），严禁假设用户环境中已预装任何非标准库。
  5. 【常量管理】代码中严禁出现裸露的“魔法数字”或硬编码字符串（如直接写 if status == 200:）。必须将其提取到文件顶部的 UPPER_SNAKE_CASE 常量中（如 HTTP_OK = 200），集中管理，方便维护。
  6. 【列表推导优先】凡是需要对列表进行筛选、映射、扁平化的操作，优先使用列表推导式（如 [x**2 for x in data if x > 0]）或生成器表达式，严禁为此专门写一个 for 循环 append。

C++

请使用 C++ 为我编写代码，运行环境为 Windows + Visual Studio Insiders。 具体需求如下： 在编写代码时，必须严格遵守以下规则：

  1. 【编码防乱码】代码中所有的打印输出和写文件内容，除注释内容外，其他地方严禁使用任何中文字符（包括中文标点），全部用纯英文代替，防止 VS 编译器出现 C4819 代码页警告或运行时乱码。
  2. 【安全函数】严禁使用 localtime、strcpy、sprintf 等被微软标记为不安全的旧函数，必须使用带 _s 后缀的安全版本（如 localtime_s），对于有返回值的函数需检查返回值，对于可能抛出异常的函数需注意安全处理。
  3. 【预处理指令】由于 #pragma execution_character_set("utf-8") 已被 VS 弃用（触发 C4996 警告），请改用以下方案之一：
    • 方案A：提示用户在项目属性中添加 /utf-8 编译选项；
    • 方案B：将源文件保存为 UTF-8 with BOM 格式，不再需要该 pragma。
  4. 【控制台防乱码】如果是控制台应用程序，在 main 函数的开头，必须调用： SetConsoleOutputCP(65001); 和 SetConsoleCP(65001);
  5. 【字符串防断行】如果代码中包含长字符串拼接，请使用 "str1" "str2" 这种断行拼接方式，避免浏览器复制时引入不可见的零宽字符导致 VS 报“常量中有换行符”错误。

HTML

请使用 HTML 为我编写代码，具体需求如下： 在编写代码时，必须严格遵守以下规则：

  1. 【字符编码】所有HTML文件必须在<!DOCTYPE html>之后、<head>标签内的第一行包含<meta charset="UTF-8">，严禁使用非UTF-8编码，防止中文或特殊字符显示乱码。
  2. 【语义化与可访问性】优先使用语义化标签（<header>, <nav>, <main>, <section>, <article>, <aside>, <footer>），为交互元素添加必要的aria-*属性或role，且所有<img>标签必须包含有意义的alt属性，确保屏幕阅读器友好。
  3. 【响应式与移动优先】必须包含 <meta name="viewport" content="width=device-width, initial-scale=1.0"> ，采用Mobile First策略，结合CSS媒体查询、Flexbox或Grid实现自适应布局，严禁出现无法在移动端正常查看的固定死宽度布局。
  4. 【样式隔离与维护性】严禁使用内联样式（style=""）和!important。样式统一写在<style>标签内，采用BEM命名规范或CSS变量（Custom Properties）进行模块化管理。（注：若明确要求使用Tailwind CSS，则直接在class中编写原子类，无需额外写<style>块）。
  5. 【JavaScript健壮性】<script>标签必须放置在</body>之前，并使用defer或type="module"属性加载。所有DOM操作必须在DOMContentLoaded事件后执行。严禁使用var，统一使用const和let避免全局污染。严禁直接插入未转义的innerHTML（防XSS），优先使用textContent或setAttribute。
  6. 【跨浏览器兼容性】若需兼容旧浏览器，在提示词中明确要求添加CSS前缀。对于现代浏览器，优先使用CSS Grid和Flexbox，针对极其边缘的旧环境可准备fallback方案（如@supports查询），但不应以牺牲现代浏览器性能为代价。
  7. 【性能优化与资源加载】图片必须设置width和height属性以避免布局偏移（CLS），非首屏图片添加loading="lazy"。CSS必须置于<head>中，JS必须置于页面底部。如需引入第三方库，优先使用外部CDN链接，但禁止引入非必要的大型框架。
  8. 【注释规范】关键布局节点、复杂业务逻辑、以及已知的浏览器兼容性hack处必须添加英文注释解释意图。严禁添加类似“作者、日期、文件名”等一次性无意义元注释。
  9. 【自包含与防占位符】生成的代码必须整合在单个.html文件中，可直接在浏览器打开并完整运行，不依赖本地服务器。严禁输出Lorem ipsum等无意义占位文本，所有文案必须使用与上下文场景高度匹配的真实内容。
  10. 【控制台零报错】确保代码在最新版Chrome/Firefox/Safari开发者工具中运行时，绝对不产生任何JavaScript错误、警告（如未捕获异常、已废弃API调用）或资源加载失败提示。

如无特殊说明，以上要求在本次对话中一直遵循。
